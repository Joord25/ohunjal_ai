## 소환 원칙 (요약)

- **영역이 명확할 때**: 해당 파일 인원 중 2-3명 한정 소환 (전원 소환은 컨텍스트 낭비)
- **영역 경계**: 러닝 프로그램 설계 → `running.md` + `data-analytics.md`(GA 이벤트 분리)
- **출처 의무**: 자문 인용 시 여기 명시된 URL/저서/논문 명시 (메모리 `feedback_source_grounded_opinions.md`)

## 추가·삭제 절차

1. 인물 추가/삭제는 실존·활동 여부·저명도 근거 3종 세트 확인
2. 파일 수정 후 메모리 `project_team_roster.md`의 해당 섹션 축약본 동기화
3. `.planning/MEETING_LOG.md`에 변경 이유 기록
